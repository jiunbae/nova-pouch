# Nova Pouch - 통합 설계서 (R01-merged)

> **"이 물건들은 이미 어딘가에 존재합니다. 당신은 그 세계를 엿볼 뿐입니다."**

---

## 0. 세계관 (Lore)

노바 파우치는 단순한 창의력 게임이 아닙니다. **현실과 평행하는 무수한 세계들이 실재하며**, 그 세계에서는 우리에게 말도 안 되는 물건들이 지극히 일상적으로 사용됩니다. 노바 파우치는 그 세계들의 파편을 담고 있는 **차원의 주머니**입니다.

플레이어는 "발명가"가 아니라 **"기록자(Archivist)"** 입니다. 주머니에서 토큰을 뽑는 행위는 무작위로 어떤 세계의 단서를 손에 쥐는 것이고, 플레이어는 그 단서를 바탕으로 **이미 존재하는 세계를 기술(describe)** 하는 것입니다.

### 핵심 서사 구조
- 3개의 주머니는 각각 **물질계(빨강)**, **성질계(파랑)**, **규율계(초록)** 에서 흘러나온 파편
- 토큰을 뽑는 것 = 세계의 단서를 발견하는 것
- 글을 쓰는 것 = 그 세계를 기록으로 남기는 것
- 히스토리 = "기록 보관소(Archive)" — 지금까지 엿본 세계들의 목록

### UI에 반영되는 톤
- "만들어보세요" (X) → **"어떤 세계가 보이시나요?"** (O)
- "창작하기" (X) → **"기록하기"** (O)
- "완성!" (X) → **"세계가 기록되었습니다"** (O)

## 1. 비주얼 디자인 시스템

> Gemini의 "우주의 서고(Cosmic Archive)" 테마를 기반으로, 세계관에 맞게 "차원의 관측소" 느낌으로 조정

### 1.1 테마 & 무드
**"Dimensional Observatory"** — 차원의 틈에서 다른 세계를 엿보는 관측소.
몽환적이면서 차분한 분위기. 어둡지만 따뜻한 빛이 스며드는 느낌.

### 1.2 컬러 팔레트
| 역할 | 색상 | HEX | 용도 |
|------|------|-----|------|
| Background | 딥 미드나잇 | `#0F0C29` → `#1A1B26` | 배경 그라데이션 |
| Red Pouch | 네뷸라 레드 | `#FF4B2B` | 물건(물질계) 주머니 |
| Blue Pouch | 에테르 블루 | `#00D2FF` | 속성(성질계) 주머니 |
| Green Pouch | 오로라 그린 | `#5DFFAB` | 제약(규율계) 주머니 |
| Accent | 스타더스트 골드 | `#FFD700` | 하이라이트, 희귀 토큰 |
| Text Primary | 라이트 | `#E8E6E3` | 본문 텍스트 |
| Text Secondary | 소프트 그레이 | `#8B8B8B` | 보조 텍스트 |

### 1.3 타이포그래피
- **UI/시스템**: `Pretendard` — 현대적이고 가독성 높은 산세리프
- **내러티브/기록**: `나눔명조` — 기록자의 서사적 분위기를 위한 세리프
- **Fallback**: 시스템 폰트 스택 (모바일 성능)

### 1.4 일러스트레이션 스타일
- 글래스모피즘 + 미세한 노이즈 텍스처
- 주머니: 반투명 유리 질감, 미세한 일렁임(Breathe) 애니메이션
- 토큰 카드: 은은한 빛(Glow) 테두리
- 색약 대응: 각 주머니에 고유 패턴(도트/스트라이프/격자) + 아이콘

## 2. 기술 아키텍처

> Claude의 모듈 구조 + Codex의 레이어 설계를 통합. MVP는 바닐라 HTML/CSS/JS.

### 2.1 파일 구조
```
nova-pouch/
├── index.html                  # SPA 엔트리
├── css/
│   ├── main.css                # CSS 변수, 리셋, 공통
│   ├── pouch.css               # 주머니 스타일 + 애니메이션
│   ├── token.css               # 토큰 카드 스타일
│   └── screens.css             # 화면별 레이아웃
├── js/
│   ├── app.js                  # 초기화, 전역 이벤트 바인딩
│   ├── state.js                # 옵저버 패턴 상태 관리
│   ├── tokens.js               # 토큰 데이터 + 뽑기 로직
│   ├── renderer.js             # DOM 렌더링, 화면 전환
│   ├── pouch.js                # 주머니 인터랙션 + 애니메이션
│   ├── history.js              # localStorage CRUD
│   └── utils.js                # 유틸리티 (RNG, 셔플 등)
└── assets/
    ├── sounds/                 # 효과음 (선택)
    └── images/                 # SVG 아이콘 (선택)
```

### 2.2 모듈 관계
```
app.js (진입점)
  ├→ state.js (상태 관리 싱글턴)
  │    ├→ subscribe/dispatch/getState
  │    └→ Phase 전환 로직
  ├→ renderer.js (화면 렌더)
  │    └→ state 변경 시 DOM 업데이트
  ├→ tokens.js (토큰 데이터)
  │    └→ 가중치 랜덤 선택 알고리즘
  ├→ pouch.js (인터랙션)
  │    └→ 클릭/탭 → state.dispatch()
  └→ history.js (저장)
       └→ localStorage 영속화
```

### 2.3 기술 결정
| 항목 | 결정 | 근거 |
|------|------|------|
| 프레임워크 | Vanilla JS (ESM) | 단순 SPA, 외부 의존성 0 |
| 스타일링 | CSS 변수 + BEM | 테마 확장, 유지보수 |
| 빌드 도구 | 없음 | 개발 복잡도 최소 |
| 상태 관리 | 커스텀 옵저버 패턴 | 경량, 충분 |
| 애니메이션 | CSS animation + JS 트리거 | 성능, GPU 가속 |
| 데이터 저장 | localStorage | 서버 불필요 |
| 반응형 | 모바일 퍼스트 | 터치 우선 설계 |

### 5.4 랜덤 선택 알고리즘
```javascript
function weightedPick(tokens, excludeId = null) {
  const candidates = excludeId
    ? tokens.filter(t => t.id !== excludeId)
    : tokens;
  const sum = candidates.reduce((a, t) => a + (t.weight || 1), 0);
  let r = Math.random() * sum;
  for (const t of candidates) {
    r -= (t.weight || 1);
    if (r <= 0) return t;
  }
  return candidates[candidates.length - 1];
}
```

## 6. 애니메이션 설계

> 3개 에이전트 합의: CSS 애니메이션 + JS 트리거 (Canvas/GSAP은 후속)

### 6.1 주머니 뽑기 시퀀스
```
[대기] idle 미세 흔들림 (3초 주기)
  │ 탭/클릭
  ▼
[흔들기] 격렬한 shake (0.5s) + 입력 차단
  │ 자동 전환
  ▼
[열기] 입구 벌어짐 (0.4s)
  │ 자동 전환
  ▼
[토큰 공개] 카드 떠오름 + 뒤집기 (0.8s)
  │ 자동 전환
  ▼
[확정 대기] [확인] / [다시 뽑기]
```

### 6.2 CSS 애니메이션
```css
@keyframes pouch-idle {
  0%, 100% { transform: rotate(0deg); }
  25% { transform: rotate(-2deg); }
  75% { transform: rotate(2deg); }
}

@keyframes pouch-shake {
  0%, 100% { transform: translateX(0); }
  10%, 30%, 50%, 70%, 90% { transform: translateX(-4px) rotate(-2deg); }
  20%, 40%, 60%, 80% { transform: translateX(4px) rotate(2deg); }
}

@keyframes token-rise {
  from { transform: translateY(100px) scale(0.5); opacity: 0; }
  to { transform: translateY(0) scale(1); opacity: 1; }
}

@keyframes token-flip {
  0% { transform: rotateY(180deg); }
  100% { transform: rotateY(0deg); }
}
```

### 6.3 성능 원칙
- `transform`, `opacity`만 애니메이션 (GPU 가속)
- `will-change`는 애니메이션 직전 부여, 직후 제거
- `prefers-reduced-motion` 시 애니메이션 즉시 건너뛰기
- 모바일: passive 이벤트 리스너

## 8. 모바일 & 접근성

### 8.1 반응형 브레이크포인트

- 최소 터치 영역 44x44px
- 햅틱 피드백: `navigator.vibrate(10)` (지원 시)

### 8.3 접근성
- ARIA 속성: `aria-live="polite"` 토큰 공개 읽기
- 색약 모드: 패턴(도트/스트라이프/격자) + 아이콘으로 주머니 구분

- 고대비 모드 옵션
- 폰트 크기 조절 (기록 화면)
